## 2024-03-27 - [A11y] Screen readers reading Material Symbol ligatures and hidden button text
**Learning:** Material Symbols use text ligatures (e.g., `<span class="material-symbols-outlined">rocket_launch</span>`). Screen readers will read the raw ligature text ("rocket launch") out loud to the user, which is confusing and non-semantic, especially if the symbol is just decorative or paired with actual text. Furthermore, some buttons in the application use responsive classes (like `hidden lg:inline`) to hide their text labels on small screens. When the text is hidden, screen reader users on mobile devices are left with no context, or just the raw ligature text.
**Action:** Always add `aria-hidden="true"` to structural icon elements (like `<span class="material-symbols-outlined">`) to prevent screen readers from reading the ligature text. For clickable elements (like `<button>` or `<a>`) that contain only icons or where the visible text is hidden responsively, ensure the parent element has an explicit `aria-label` attribute to maintain consistent and meaningful context for screen readers across all viewport sizes.
